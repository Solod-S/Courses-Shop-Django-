# CSS / Frontend Style Guide & Workspace Rules

## 1. Where to Store Styles

```
shop/
  static/
    shop/
      css/
        courses.css       # Styles for one specific page/template
        base.css          # Global reset, variables, typography
        components.css    # Reusable component styles (buttons, cards, etc.)
        ...               # One file per logical group
      js/                 # JavaScript files (if needed)
      img/                # Images
```

**Rule:** Always place static files inside `shop/static/shop/`. Django automatically discovers static files in each app's `static/` directory (thanks to `django.contrib.staticfiles`).

- **One CSS file per page / logical block** — don't put everything in one file.
- Name the CSS file after the template it styles: `courses.html` → `courses.css`
- Global/ shared styles → `base.css`

---

## 2. How to Link CSS in Templates

```django
{% load static %}
<!doctype html>
<html lang="en">
  <head>
    <link rel="stylesheet" href="{% static 'shop/css/courses.css' %}" />
  </head>
```

Always use `{% static %}` tag — never hardcode `/static/...`.

## 3. CSS Methodology → BEM

Use **BEM** (Block Element Modifier) for all class names.

```
.block {}
.block__element {}
.block--modifier {}
```

**Examples:**

```css
.course-table {
} /* Block */
.course-table__header {
} /* Element */
.course-table__row--active {
} /* Modifier */
.product-card {
} /* Block */
.product-card__title {
} /* Element */
.product-card__price--sale {
} /* Modifier */
```

**Why BEM:**

- No nesting conflicts
- Predictable naming
- Easy to maintain

---

## 4. Naming Conventions

| Thing                 | Convention                  | Example                           |
| --------------------- | --------------------------- | --------------------------------- |
| CSS class names       | kebab-case                  | `.course-table`, `.btn-primary`   |
| CSS custom properties | kebab-case with `--` prefix | `--color-primary`, `--spacing-md` |
| File names            | kebab-case                  | `courses.css`, `base.css`         |
| HTML data attributes  | kebab-case                  | `data-course-id`, `data-action`   |

**No camelCase or snake_case in CSS class names.**

---

## 5. Design Philosophy — Strict, Minimal, Professional

**No gradients. No aggressive shadows. No decorative emoji.**

- Background: neutral light gray (`#f5f6f8`)
- Surfaces: clean white (`#ffffff`) with subtle `1px solid #e0e0e0` borders
- Text: near-black (`#1d1d1f`) and muted gray (`#6e6e73`)
- Border radius: small — `6px` or `8px` (never `16px`)
- Shadows: minimal or none
- Buttons: solid dark fill (`#1d1d1f`), no gradient, no lift-on-hover effects

---

## 6. Responsive Design

- **Mobile-first** approach.
- Use `min-width` media queries (not `max-width`).

```css
/* Base (mobile) styles first */
.course-table {
  font-size: 0.85rem;
}

/* Tablet */
@media (min-width: 768px) {
  .course-table {
    font-size: 0.95rem;
  }
}

/* Desktop */
@media (min-width: 1024px) {
  .course-table {
    font-size: 1rem;
  }
}
```

- Use `rem` / `em` for sizes, `px` only for borders.
- Use `%` / `vw` / `fr` for layout widths.
- Use `clamp()` for fluid typography when needed:

```css
font-size: clamp(0.85rem, 2vw, 1.1rem);
```

## 7. Color Palette (Custom Properties)

Define all colors in `base.css` as CSS custom properties on `:root`:

```css
:root {
  /* Neutral palette — strict & minimal */
  --color-bg: #f5f6f8;
  --color-surface: #ffffff;
  --color-border: #e0e0e0;
  --color-border-light: #f0f0f0;

  --color-text: #1d1d1f;
  --color-text-muted: #6e6e73;

  /* Interactive */
  --color-btn-primary: #1d1d1f;
  --color-btn-primary-hover: #3a3a3c;
  --color-btn-text: #ffffff;

  /* Table */
  --color-table-header-bg: #fafafa;
  --color-table-header-border: #e0e0e0;
  --color-table-row-hover: #fafafa;

  /* Typography */
  --font-family-base:
    -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen, Ubuntu,
    Cantarell, sans-serif;
  --font-size-base: 0.9rem;
  --font-size-sm: 0.78rem;
  --font-size-xs: 0.72rem;
  --font-size-lg: 1.1rem;
  --font-weight-normal: 400;
  --font-weight-medium: 500;
  --font-weight-semibold: 600;

  /* Spacing (4px base) */
  --spacing-xs: 0.25rem;
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;
  --spacing-lg: 1.5rem;
  --spacing-xl: 2rem;

  /* Border radius — small, sharp */
  --radius-sm: 4px;
  --radius-md: 6px;
  --radius-lg: 8px;

  /* Shadows — minimal */
  --shadow-none: none;
  --shadow-subtle: 0 1px 3px rgba(0, 0, 0, 0.06);

  /* Transitions — quick, subtle */
  --transition-fast: 0.15s ease;
  --transition-base: 0.2s ease;
}
```

**Never hardcode raw hex values in component CSS** — always reference a variable.

---

## 8. Typography

- Font stack: system fonts (`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, ...`)
- Base font size: `--font-size-base` on `body`.
- Use relative units (`rem`, `em`) for all font sizes.
- Line-height: `1.5` for body, `1.3` for headings.
- Color: `--color-text` for primary text, `--color-text-muted` for secondary text.

---

## 9. Spacing

- Use the `--spacing-*` custom properties.
- Margins, paddings, gaps — never arbitrary px values.

```css
/* ✅ Good */
.card {
  padding: var(--spacing-md);
  margin-bottom: var(--spacing-lg);
  gap: var(--spacing-sm);
}

/* ❌ Bad */
.card {
  padding: 13px;
  margin-bottom: 19px;
}
```

---

## 10. Layout

- Use **Flexbox** or **CSS Grid** — never floats for layout.
- Use `gap` instead of margin hacks.

```css
.page-section {
  width: 100%;
  max-width: 960px;
  margin: 0 auto;
}
```

Tables that overflow should use `overflow-x: auto` on a wrapper element, with `-webkit-overflow-scrolling: touch` for mobile smoothness. The table itself gets `min-width: XXpx` to trigger the scroll on small viewports.

```css
.table-wrapper {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
}
table {
  min-width: 600px;
}
```

---

## 11. Tables

- Use `border-collapse: collapse`.
- Header: light gray background (`--color-table-header-bg`), bold text, `border-bottom`.
- Rows: separated by `1px solid --color-border-light`.
- Row hover: subtle `--color-table-row-hover` background.
- `white-space: nowrap` on cells to prevent wrapping during horizontal scroll.

```css
thead th {
  background: var(--color-table-header-bg);
  padding: 0.85rem 1rem;
  font-weight: var(--font-weight-semibold);
  font-size: var(--font-size-xs);
  border-bottom: 2px solid var(--color-table-header-border);
  white-space: nowrap;
}
tbody td {
  padding: 0.85rem 1rem;
  white-space: nowrap;
}
tbody tr:hover {
  background-color: var(--color-table-row-hover);
}
```

---

## 12. Buttons

- Solid dark fill (`--color-btn-primary`), no gradient, no shadow.
- Border-radius: `--radius-md` (6px).

```css
button {
  background: var(--color-btn-primary);
  color: var(--color-btn-text);
  border: none;
  padding: 0.4rem 1.2rem;
  border-radius: var(--radius-md);
  font-size: var(--font-size-sm);
  font-weight: var(--font-weight-medium);
  cursor: pointer;
  transition: background-color var(--transition-fast);
  font-family: inherit;
}
button:hover {
  background: var(--color-btn-primary-hover);
}
```

---

## 13. Transitions & Animations

- Keep them subtle and fast — `0.15s` or `0.2s`.
- Only animate `background-color`, `opacity`, `transform`.
- No decorative animations (no bounce, no float, no glow).

---

## 14. Order of CSS Properties

Within a rule, order properties logically:

```
1. Positioning       (position, top, z-index)
2. Display / Layout  (display, flex, grid, gap)
3. Box model         (width, height, margin, padding)
4. Background        (background, color)
5. Border            (border, border-radius, box-shadow)
6. Typography        (font-*, line-height, text-*)
7. Other             (cursor, overflow, transition)
```

---

## 15. File Template

When creating a new CSS file, always start with this:

```css
/* ===== [Block Name] Styles ===== */

/* Desktop-first base styles */
...

/* Tablet (max-width: 768px) */
...

/* Mobile (max-width: 480px) */
...
```

---

## 16. Formatting & Linting

- 2 spaces indentation (no tabs).
- One selector per line in multi-selector rules.
- One space before `{`.
- One space after `:`.
- Semicolons after every declaration (including the last).
- Blank line between rules.

```css
/* ✅ Good */
.card,
.card--highlighted {
  padding: var(--spacing-md);
  background: var(--color-surface);
}

/* ❌ Bad */
.card,
.card--highlighted {
  padding: var(--spacing-md);
  background: var(--color-surface);
}
```

---

## 17. Accessibility

- Always use relative units so text can be resized by the browser.
- Ensure contrast ratio ≥ 4.5:1 for normal text.
- Interactive elements (buttons, links) must have visible `:focus` styles.

---

## 18. Do NOT

- ❌ Use `!important` (unless overriding a 3rd-party library — then comment why).
- ❌ Use gradients, decorative shadows, emoji icons, or colorful accents.
- ❌ Hardcode colors / sizes — always use CSS custom properties.
- ❌ Use inline styles — all styles go in CSS files.
- ❌ Use `id` selectors for styling — use classes only.
- ❌ Nest selectors deeper than 2 levels (BEM avoids this naturally).
- ❌ Use `px` for font-sizes (use `rem`) or spacing (use `rem` via custom properties).
